# 项目架构说明

## 整体设计

本项目采用模块化设计，每个lab都是独立的可运行示例，同时共享一些通用工具库。

## 目录结构说明

### docs/
存放项目文档，包括：
- 整体架构说明（本文档）
- 各主题的详细文档（topics/）

### pkg/
共享库，提供可复用的工具：
- `storage/`: 存储相关工具（如文件操作、序列化等）
- `network/`: 网络通信工具（用于分布式demo）
- `utils/`: 通用工具函数

### labs/
所有实验demo的目录，按主题组织：
- 每个lab都是独立的Go程序
- 可以单独运行和测试
- 包含README说明文档

## 设计原则

1. **独立性**: 每个lab可以独立运行，不依赖其他lab
2. **可理解性**: 代码简洁清晰，注释充分
3. **可测试性**: 每个lab都包含测试用例
4. **教育性**: 重点展示概念和权衡，而非追求性能

## 技术选型

- **语言**: Go 1.21+
- **测试框架**: 标准testing包 + testify
- **网络通信**: 标准库net/http（简单场景）或gRPC（复杂场景）
- **序列化**: encoding/json（简单场景）或protobuf（性能要求高时）
